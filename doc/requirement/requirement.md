## 1. 需求背景和目标

### 1.1 背景概述

SKILL管理平台是公司内部SKILL资源的集中管理载体，用于管理公共广场SKILL、套件SKILL、个人未发布SKILL三类资源，实现"上传-校验-存储-检索-下载-部署-维护"的全流程闭环管理。

平台需满足不同角色用户的使用需求：
- 普通用户：上传、下载、使用SKILL
- 管理员：全局管控、审核、用户管理
- 开发者：管理自身SKILL、参与公共SKILL维护

### 1.2 用户角色和场景

| 角色 | 核心场景 |
|------|----------|
| 普通用户 | 上传个人SKILL、下载公共/套件SKILL、一键部署、检索SKILL、收藏/评价SKILL |
| SKILL开发者 | 提交公共SKILL审核、创建/维护套件SKILL、管理SKILL版本、查看使用反馈 |
| 普通管理员 | 审核SKILL、管理用户、管理系统配置 |
| 超级管理员 | 全部管理功能、权限分配、系统配置 |

### 1.3 核心痛点

- **痛点1**: SKILL资源分散，缺乏统一管理
  - 影响范围: 全公司用户
  - 严重程度: 高

- **痛点2**: SKILL格式不规范，质量参差不齐
  - 影响范围: SKILL使用者
  - 严重程度: 高

- **痛点3**: SKILL部署流程复杂，使用门槛高
  - 影响范围: 普通用户
  - 严重程度: 中

- **痛点4**: 缺乏SKILL关联关系可视化，难以规划学习路径
  - 影响范围: SKILL学习者
  - 严重程度: 中

### 1.4 用户旅程

```
用户登录 → 检索SKILL → 查看详情 → 下载/部署 → 使用SKILL → 评价反馈
    ↓
开发者上传 → 格式校验 → 填写信息 → 提交审核 → 发布上线 → 版本迭代
```

### 1.5 需求目标

- **目标1**: 实现SKILL资源的集中管理 - 成功标准: 支持三种类型SKILL的全生命周期管理
- **目标2**: 提供严格的格式校验机制 - 成功标准: 校验通过率≥95%，误报率≤1%
- **目标3**: 实现SKILL一键部署 - 成功标准: 部署响应时间≤10秒，成功率≥99%
- **目标4**: 建立完善的权限管理体系 - 成功标准: 无越权访问事件
- **目标5**: 提供SKILL图谱功能 - 成功标准: 支持SKILL关联关系可视化展示

#### 2.2.1 接口功能

##### 功能F001: 用户认证服务 (全新功能)

###### 功能描述

**功能类型**: 全新功能
**功能实现类型**: 接口功能
**业务目标**: 提供用户登录、注册、密码管理、会话管理功能，确保用户身份合法
**技术目标**: 实现基于JWT的认证机制，支持账号密码登录和OAuth2/OIDC单点登录

###### 输入定义

**登录接口**:

| 输入项 | 类型 | 来源 | 必填 | 约束条件 | 示例 |
|-------|------|------|------|---------|------|
| username | String | 请求参数 | 是 | 工号，长度6-20位 | "EMP001" |
| password | String | 请求参数 | 是 | 长度8-50位 | "P@ssw0rd123" |
| loginType | String | 请求参数 | 否 | 枚举: password, sso | "password" |

**输入校验规则**:
1. username: 必须为公司工号格式，仅允许字母和数字
2. password: 长度8-50位，必须包含字母、数字、特殊符号
3. loginType: 默认为password

**注册接口**:

| 输入项 | 类型 | 来源 | 必填 | 约束条件 | 示例 |
|-------|------|------|------|---------|------|
| employeeId | String | 请求参数 | 是 | 公司工号，唯一 | "EMP001" |
| name | String | 请求参数 | 是 | 长度2-50位 | "张三" |
| department | String | 请求参数 | 是 | 部门名称 | "技术部" |
| email | String | 请求参数 | 是 | 公司邮箱格式 | "zhangsan@company.com" |
| password | String | 请求参数 | 是 | 长度8-50位，包含字母、数字、特殊符号 | "P@ssw0rd123" |

**输入校验规则**:
1. employeeId: 必须唯一，不可与已有用户重复
2. email: 必须为公司邮箱域名
3. password: 必须符合复杂度要求

###### 输出定义

**登录成功响应**:
```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "expiresIn": 7200,
    "userInfo": {
      "id": "user001",
      "employeeId": "EMP001",
      "name": "张三",
      "department": "技术部",
      "role": "USER"
    }
  }
}
```

**失败响应**:

| 错误码 | 错误信息 | 触发条件 | 处理建议 |
|-------|---------|---------|---------|
| AUTH001 | 用户名或密码错误 | 密码验证失败 | 重新输入 |
| AUTH002 | 账号已被锁定 | 连续5次登录失败 | 联系管理员解锁 |
| AUTH003 | 账号未审核 | 注册后未通过审核 | 等待管理员审核 |
| AUTH004 | 账号已被禁用 | 管理员禁用账号 | 联系管理员 |
| AUTH005 | SSO认证失败 | OAuth2/OIDC认证失败 | 检查SSO配置 |

###### 边界条件处理

| 边界条件 | 处理方式 |
|---------|---------|
| 连续5次登录失败 | 锁定账号30分钟，记录安全日志 |
| Token过期 | 返回401，客户端使用refreshToken刷新 |
| refreshToken过期 | 返回401，要求重新登录 |
| 并发登录 | 支持多设备同时登录，最多5个设备 |
| SSO回调超时 | 返回错误提示，引导重新登录 |

**技术目标**: 实现角色-权限映射，支持细粒度的资源访问控制

**角色查询接口**:

| 输入项 | 类型 | 来源 | 必填 | 约束条件 | 示例 |

**权限验证接口**:

| userId | String | 请求参数 | 是 | 用户ID | "user001" |

| 边界条件 | 处理方式 |

| 资源权限继承 | 套件SKILL权限继承至子SKILL |
| 跨部门访问 | 默认拒绝，需显式授权 |

##### 功能F003: SKILL上传服务 (全新功能)

###### 功能描述

**功能类型**: 全新功能
**功能实现类型**: 接口功能
**业务目标**: 支持用户上传SKILL文件，包括单文件上传、批量上传、拖拽上传
**技术目标**: 实现文件分片上传，支持断点续传，最大支持100MB文件

###### 输入定义

**单文件上传接口**:

| 输入项 | 类型 | 来源 | 必填 | 约束条件 | 示例 |
|-------|------|------|------|---------|------|
| file | File | 表单数据 | 是 | 支持.json/.skill/.zip，最大100MB | - |
| skillType | String | 请求参数 | 是 | 枚举: public, suite, personal | "public" |
| name | String | 请求参数 | 是 | 长度1-100位 | "数据统计SKILL" |
| description | String | 请求参数 | 否 | 最大500字 | "用于数据统计分析" |
| version | String | 请求参数 | 是 | 格式: x.y.z | "1.0.0" |
| category | String | 请求参数 | 是 | SKILL分类 | "办公类" |
| tags | Array | 请求参数 | 否 | 标签列表 | ["python", "data"] |

**输入校验规则**:
1. file: 格式必须为.json/.skill/.zip，大小≤100MB
2. name: 不得包含特殊字符，长度1-100位
3. version: 必须符合语义化版本规范(x.y.z)
4. skillType: 必须为枚举值之一

**批量上传接口**:

| 输入项 | 类型 | 来源 | 必填 | 约束条件 | 示例 |
|-------|------|------|------|---------|------|
| files | File[] | 表单数据 | 是 | 最多10个文件 | - |
| skillType | String | 请求参数 | 是 | 统一SKILL类型 | "personal" |

###### 输出定义

**上传成功响应**:
```json
{
  "code": 200,
  "message": "上传成功",
  "data": {
    "skillId": "skill001",
    "name": "数据统计SKILL",
    "version": "1.0.0",
    "status": "pending_validation",
    "uploadTime": "2026-05-16T10:30:00Z"
  }
}
```

**失败响应**:

| 错误码 | 错误信息 | 触发条件 | 处理建议 |
|-------|---------|---------|---------|
| UPLOAD001 | 文件格式不支持 | 文件格式不在允许列表 | 上传.json/.skill/.zip文件 |
| UPLOAD002 | 文件大小超限 | 文件>100MB | 压缩文件或拆分SKILL |
| UPLOAD003 | 文件名不规范 | 文件名包含中文/空格/特殊字符 | 修改文件名 |
| UPLOAD004 | 存储空间不足 | 个人存储空间已满 | 删除无用SKILL |
| UPLOAD005 | 版本号已存在 | 同名SKILL版本号重复 | 使用新版本号 |

###### 边界条件处理

| 边界条件 | 处理方式 |
|---------|---------|
| 网络中断 | 支持断点续传，保留已上传分片 |
| 重复上传 | 提示"文件已存在"，可选择覆盖或取消 |
| 并发上传 | 支持同时上传多个文件，互不影响 |
| 批量上传部分失败 | 记录失败文件，成功文件正常处理 |

**功能类型**: 全新功能
**功能实现类型**: 接口功能
**业务目标**: 支持用户下载SKILL文件，包括单文件下载、批量下载、版本选择下载
**技术目标**: 实现文件流式下载，支持断点续传

###### 输入定义

**单文件下载接口**:

| 输入项 | 类型 | 来源 | 必填 | 约束条件 | 示例 |

##### 功能F004: SKILL格式校验服务 (全新功能)

###### 功能描述

**功能类型**: 全新功能
**功能实现类型**: 后台功能
**业务目标**: 对上传的SKILL文件进行严格的格式校验，确保文件规范、安全
**技术目标**: 实现多维度校验（格式、命名、内容、安全），支持离线CLI校验

**校验规则**:

1. **文件格式校验**:
   - 支持格式: .json, .skill, .zip
   - 编码要求: UTF-8，无BOM头
   - 校验方式: 文件扩展名+魔数校验

2. **文件命名校验**:
   - 不得包含: 空格、中文、特殊字符(@#$%^&等)
   - 不得包含: ".."、连续斜杠
   - 长度限制: 1-255字符
   - 保留文件名: CON, PRN, AUX, NUL, COM1-9, LPT1-9

3. **文件内容校验**:
   - JSON格式: 语法校验、结构校验、核心字段校验
   - ZIP格式: 完整性校验、核心文件存在性校验
   - 核心字段: name, version, description, triggers, steps

4. **版本格式校验**:
   - 格式: x.y.z (x, y, z为非负整数)
   - 示例: 1.0.0, 2.1.3

5. **大小校验**:
   - 单文件: ≤100MB
   - 批量总大小: ≤500MB
   - 个人存储: ≤1GB

6. **安全校验**:
   - 恶意代码检测
   - 敏感信息检测
   - 依赖漏洞扫描
   - 具体规则待设计阶段确定

**校验流程**:
```
文件上传 → 格式校验 → 命名校验 → 内容校验 → 版本校验 → 大小校验 → 安全校验 → 校验结果
```

**校验结果**:
- 通过: 返回校验通过，继续上传流程
- 失败: 返回失败原因列表，包含字段位置和错误描述
