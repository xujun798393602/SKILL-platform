# SKILL管理平台详细需求文档

# 一、文档概述

## 1\.1 文档目的

本文档旨在明确SKILL管理平台的详细需求，涵盖功能、性能、易用性、安全性等多个维度，为平台的设计、开发、测试及部署提供标准化依据，确保平台能够满足公司对各类SKILL（公共广场SKILL、套件SKILL、个人未发布SKILL）的全生命周期管理需求，实现SKILL的便捷上传、下载、格式校验及一键部署，提升公司SKILL资源的复用率和管理效率。

## 1\.2 适用范围

本文档适用于SKILL管理平台的产品设计、开发人员、测试人员、运维人员及所有平台使用用户（包括普通员工、管理员、SKILL开发者），作为各方协同工作的核心参考依据。

## 1\.3 核心目标

1. 实现公司各类SKILL的集中管理，区分公共、套件、个人未发布三种类型，清晰划分权限与可见范围；

2. 支持SKILL的便捷上传与下载，具备严格的格式校验机制，确保SKILL文件的规范性和可用性；

3. 提供SKILL一键部署功能，简化部署流程，降低使用门槛，提升SKILL落地效率；

4. 建立完善的权限管理体系，保障SKILL资源的安全性，防止未授权访问、篡改或泄露；

5. 提供SKILL检索、分类、版本管理等辅助功能，提升SKILL的查找和复用效率；

6. 支持系统可扩展性，便于后续新增功能、适配更多SKILL格式及集成其他业务系统。

# 二、总体业务需求

SKILL管理平台是公司内部SKILL资源的集中管理载体，核心用于管理公共广场SKILL、套件SKILL、个人未发布SKILL三类资源，实现“上传\-校验\-存储\-检索\-下载\-部署\-维护”的全流程闭环管理。平台需满足不同角色用户的使用需求，普通用户可上传、下载、使用SKILL，管理员可进行全局管控，开发者可管理自身SKILL并参与公共SKILL的维护，同时确保系统稳定、高效、安全运行，符合企业级应用的使用标准。

平台整体遵循“分类管理、规范校验、便捷操作、安全可控”的原则，参考企业级技能管理平台的设计理念，兼顾易用性与专业性，支持自托管部署确保数据主权完全自控，同时提供友好的Web交互界面与灵活的操作方式，降低用户使用成本。

# 三、详细功能需求

## 3\.1 基础功能模块

### 3\.1\.1 用户登录与注册

1. 登录功能：支持公司内部账号密码登录、单点登录（SSO）两种方式，登录失败时给出明确提示（如密码错误、账号锁定），连续多次登录失败后锁定账号，需管理员解锁；

2. 注册功能：仅支持公司内部员工注册，需填写工号、姓名、部门、邮箱等信息，注册后需管理员审核通过方可使用平台；

3. 密码管理：支持密码重置（通过邮箱验证）、密码修改，密码需符合复杂度要求（如8位以上、包含字母、数字、特殊符号）；

4. 会话管理：用户登录后生成会话，超时未操作自动退出登录，支持手动退出；

5. 个人中心：用户可查看个人信息（工号、姓名、部门等），修改个人资料、密码，查看个人SKILL操作记录（上传、下载、部署）。

### 3\.1\.2 权限管理

平台采用基于命名空间的RBAC权限体系，分为超级管理员、普通管理员、普通用户、SKILL开发者四种角色，不同角色权限严格区分，确保资源安全可控，具体权限如下：

1. 超级管理员：拥有平台全部操作权限，包括用户管理、角色分配、SKILL审核、系统配置、日志查看、权限分配等；

2. 普通管理员：负责SKILL审核（公共广场SKILL、套件SKILL）、用户管理（查看、禁用普通用户）、SKILL分类管理、系统基础配置；

3. 普通用户：可上传个人SKILL（未发布至广场）、下载公共广场SKILL和已授权的套件SKILL、使用SKILL、检索SKILL、查看个人SKILL；

4. SKILL开发者：在普通用户权限基础上，可提交公共广场SKILL、参与套件SKILL的创建与维护、查看SKILL使用反馈、管理SKILL版本。

权限细化：支持对单个SKILL设置访问权限（如个人可见、部门可见、全公司可见），套件SKILL可设置子SKILL的访问权限，确保SKILL资源的精准管控。

## 3\.2 SKILL分类管理功能

平台将SKILL分为三大类，分别进行差异化管理，同时支持SKILL分类的自定义配置，适配公司业务发展需求，各类SKILL的管理规则如下：

### 3\.2\.1 公共广场SKILL

1. 定义：由SKILL开发者提交、经管理员审核通过后，发布至公共广场，供全公司所有用户查看、下载、使用的SKILL，具备通用性和复用性；

2. 发布流程：开发者上传SKILL→填写SKILL信息（名称、描述、版本、分类、适用场景等）→提交审核→管理员审核（审核内容包括格式合规性、内容安全性、可用性）→审核通过后发布至公共广场，审核失败则反馈失败原因，开发者可修改后重新提交；

3. 管理操作：管理员可编辑、下架、删除公共广场SKILL，开发者可修改已提交但未审核的SKILL，审核通过后需申请管理员权限方可修改；

4. 可见范围：全公司所有已登录用户可见，支持检索、筛选、排序（按热度、发布时间、下载量）；

5. 附加功能：支持用户对公共广场SKILL进行评分、评论、收藏，开发者可查看使用反馈，便于SKILL优化迭代；支持语义化版本控制，同一个SKILL可保留多个历史版本并存，用户可选择下载对应版本。

### 3\.2\.2 套件SKILL

1. 定义：由多个相关联的SKILL（可包含公共SKILL、个人SKILL）组合而成，用于完成特定业务场景的一系列操作，如“客户管理套件”“数据统计套件”；

2. 创建与编辑：由SKILL开发者或管理员创建，可添加、删除、调整套件内的SKILL，设置套件的名称、描述、适用场景、版本、访问权限（如部门可见、全公司可见）；

3. 审核流程：套件SKILL创建后，需提交管理员审核，审核内容包括套件内SKILL的合规性、关联性、可用性，审核通过后可发布使用，审核失败反馈修改意见；

4. 使用与管理：授权用户可下载整个套件SKILL，也可单独下载套件内的单个SKILL；管理员可下架、删除套件SKILL，开发者可修改已创建但未审核的套件，审核通过后需申请管理员权限修改；

5. 版本管理：支持套件SKILL的版本迭代，每次修改后生成新的版本，保留历史版本，用户可选择下载对应版本的套件，同时支持自定义latest、beta、stable等版本标签，便于用户区分版本类型。

### 3\.2\.3 个人未发布SKILL

1. 定义：由用户上传，仅本人可见、可操作，未提交至公共广场、未加入套件的SKILL，用于个人工作使用或临时存储；

2. 上传与管理：用户可自由上传个人SKILL，填写基本信息（名称、描述、版本、分类），无需审核，可自行编辑、删除、修改SKILL信息及内容；

3. 转化功能：用户可将个人未发布SKILL提交至公共广场（需经过管理员审核），或添加至已创建的套件SKILL中（需具备套件编辑权限）；

4. 可见范围：仅上传用户本人可见，管理员无法查看个人未发布SKILL（除非用户主动提交审核或添加至套件）；

5. 存储管理：个人SKILL存储有容量限制，超出限制后无法上传新的SKILL，用户可删除无用SKILL释放容量，管理员可设置全局个人SKILL存储容量标准。

## 3\.3 SKILL上传与下载功能

### 3\.3\.1 SKILL上传

1. 上传方式：支持本地文件上传（单次上传单个或多个SKILL文件）、拖拽上传，同时支持文件夹上传，无需强制要求特定根文件，提升上传灵活性；

2. 上传要求：用户需选择SKILL类型（公共、套件、个人），填写SKILL基本信息（名称、描述、版本、分类、适用场景、依赖环境等），套件SKILL还需选择包含的SKILL；

3. 格式限制：支持常见的SKILL文件格式（如\.json、\.skill、\.zip等），管理员可在系统配置中添加或删除支持的格式，同时明确禁止上传的文件格式；

4. 进度提示：上传过程中显示上传进度（百分比），上传成功、失败均给出明确提示，失败时说明原因（如格式不支持、文件过大、网络异常）；

5. 批量上传：支持多个SKILL文件批量上传，批量上传时需统一选择SKILL类型，单个文件上传失败不影响其他文件的上传进度。

### 3\.3\.2 SKILL下载

1. 下载方式：支持单个SKILL下载、多个SKILL批量下载、套件SKILL整体下载，单个文件SKILL可直接下载，多文件SKILL自动压缩为ZIP包下载，无需额外打包操作；

2. 权限控制：仅有权限的用户可下载对应SKILL（公共广场SKILL全公司可下载，套件SKILL仅授权用户可下载，个人SKILL仅本人可下载）；

3. 版本选择：支持选择SKILL的历史版本进行下载，默认下载最新版本；

4. 下载提示：下载开始时给出提示，下载成功后提示文件保存路径（浏览器默认路径或用户自定义路径），下载失败说明原因（如权限不足、文件不存在、网络异常）；

5. 下载记录：系统记录用户的SKILL下载记录（下载时间、SKILL名称、版本、大小），用户可在个人中心查看自己的下载记录。

## 3\.4 SKILL格式校验功能

格式校验是保障SKILL可用性的核心环节，平台需对上传的所有SKILL进行严格的格式校验，校验失败的SKILL无法完成上传或提交审核，具体校验规则如下：

1. 文件格式校验：校验上传文件的格式是否为平台支持的格式，禁止上传不支持格式的文件，同时校验文件编码（需为UTF\-8编码、无BOM头）；

2. 文件命名校验：文件名不得包含空格、中文字符或特殊符号（如@、\#、$、%等），不得为空，不得包含“\.\.”、连续斜杠，不得使用系统保留文件名，长度不得超过规定限制；

3. 文件内容校验：
        

    - 结构化文件（如\.json、\.skill）：校验文件语法是否正确、结构是否完整，核心字段（如名称、描述、触发条件、执行步骤）是否缺失，字段类型是否符合要求；

    - 压缩文件（如\.zip）：校验压缩包是否完整、无损坏，解压后是否包含SKILL核心文件，压缩包大小是否符合限制；

4. 版本格式校验：校验SKILL版本号是否符合规范（如x\.y\.z格式，x、y、z为非负整数），避免版本号混乱；

5. 大小校验：单个SKILL文件大小、批量上传总大小、个人SKILL存储总大小均需符合系统配置的限制，超出限制则校验失败；

6. 安全性校验：内置安全扫描机制，对上传的SKILL文件进行多引擎安全分析，包括恶意代码检测、敏感信息泄露、依赖漏洞扫描和行为分析等，扫描异常则校验失败并提示风险；

7. 校验反馈：校验失败时，明确提示失败原因（如“文件格式不支持，请上传\.json格式文件”“核心字段缺失，需补充描述字段”“文件包含敏感信息，禁止上传”），用户可根据提示修改后重新上传；

8. 高级校验：支持通过CLI工具进行离线结构校验，适用于批量验证或CI/CD集成场景，输出精确错误定位（如缺失字段位置、非法字符偏移量），便于开发者快速排查问题。

## 3\.5 SKILL一键部署功能

一键部署功能旨在简化SKILL的落地流程，降低使用门槛，支持公共广场SKILL、套件SKILL、个人SKILL的一键部署，参考llamafile一键部署的设计理念，实现SKILL的快速启动与运行，具体功能如下：

1. 部署前提：SKILL需通过格式校验，且用户具备部署权限（个人SKILL仅本人可部署，公共、套件SKILL全公司有权限用户可部署）；

2. 部署操作：用户在SKILL详情页点击“一键部署”按钮，系统自动完成SKILL的部署流程，无需用户手动配置依赖环境；

3. 部署配置：支持简单的部署参数配置（如部署路径、运行端口、启动方式等），默认使用系统推荐配置，用户可根据需求修改；

4. 部署状态反馈：部署过程中显示部署进度（如“正在部署中\.\.\.30%”），部署成功、失败均给出明确提示，失败时说明原因（如“依赖环境缺失”“端口被占用”“权限不足”）；

5. 部署后管理：
        

    - 部署成功后，用户可查看SKILL的运行状态（运行中、停止、异常）；

    - 支持一键停止、重启部署的SKILL，停止后可重新部署；

    - 记录SKILL的部署记录（部署时间、部署用户、部署参数、运行状态），用户可在个人中心或SKILL详情页查看；

6. 跨平台支持：支持Windows、macOS、Linux等多种操作系统的一键部署，无需用户针对不同系统进行额外配置，实现跨平台兼容；

7. 批量部署：支持多个SKILL或整个套件SKILL的批量一键部署，批量部署时可统一配置部署参数，单个SKILL部署失败不影响其他SKILL的部署。

## 3\.6 SKILL检索与筛选功能

1. 检索方式：支持关键词检索（检索SKILL名称、描述、分类、适用场景等）、高级检索（按SKILL类型、版本、上传时间、开发者、下载量、评分等条件组合检索）；

2. 检索优化：检索结果支持模糊匹配，检索速度快（响应时间≤1秒），支持检索历史记录保存，用户可快速重新检索；

3. 筛选功能：支持按SKILL类型（公共、套件、个人）、分类（如办公类、开发类、业务类）、版本、上传时间、下载量、评分等条件筛选，筛选结果可排序（升序、降序）；

4. 标签筛选：支持为SKILL添加标签，用户可通过标签快速筛选相关SKILL，管理员可统一管理标签体系，确保标签的规范性；

5. 结果展示：检索、筛选结果以列表形式展示，显示SKILL核心信息（名称、类型、版本、开发者、下载量、评分、发布时间），点击可进入SKILL详情页。

## 3\.7 SKILL版本管理功能

1. 版本生成：SKILL上传后自动生成初始版本（如1\.0\.0），后续修改SKILL内容或信息后，可手动升级版本（如1\.0\.1、1\.1\.0），版本号需符合规范；

2. 版本记录：系统保留SKILL的所有历史版本，记录每个版本的修改时间、修改人、修改内容、版本号，用户可查看历史版本详情；

3. 版本切换：支持在不同版本之间切换，查看不同版本的SKILL内容，可下载历史版本、部署历史版本；

4. 版本回滚：当当前版本出现问题时，用户可将SKILL回滚至之前的历史版本，回滚后生成新的版本记录，保留原版本内容；

5. 版本备注：支持为每个版本添加备注（如“修复了XX问题”“新增了XX功能”），便于用户区分不同版本的差异；

6. 版本标签：支持自定义版本标签（如latest、beta、stable），用户可快速识别稳定版本、测试版本，管理员可设置标签的使用规则。

## 3\.8 系统管理功能

### 3\.8\.1 用户管理

1. 用户列表：管理员可查看平台所有用户信息（工号、姓名、部门、角色、账号状态、注册时间）；

2. 用户操作：管理员可新增用户、编辑用户信息、修改用户角色、禁用/启用用户账号，删除无用用户账号；

3. 批量操作：支持批量导入、导出用户信息，批量修改用户角色、禁用/启用用户账号；

4. 账号监控：管理员可查看用户的登录记录（登录时间、登录IP、登录设备），及时发现异常登录行为。

### 3\.8\.2 系统配置

1. 基础配置：设置平台名称、logo、公告、系统版本等基础信息；

2. SKILL配置：设置支持的SKILL文件格式、单个SKILL文件大小限制、批量上传大小限制、个人SKILL存储容量限制，添加或删除禁止上传的文件格式；

3. 权限配置：修改各角色的权限范围，新增自定义角色并分配权限，配置命名空间及成员权限；

4. 部署配置：设置SKILL默认部署参数（如默认部署路径、运行端口），配置部署环境的相关参数，支持Docker Compose、K8s等多种部署方式的配置；

5. 安全配置：设置密码复杂度要求、账号锁定规则、会话超时时间，配置安全扫描参数，设置敏感信息检测规则；

6. 备份配置：设置系统数据备份周期（如每日备份、每周备份）、备份路径，支持手动备份、自动备份，备份数据可恢复。

### 3\.8\.3 日志管理

1. 日志类型：记录平台所有操作日志（用户登录、SKILL上传/下载/部署/修改/删除、权限变更、系统配置修改等）、错误日志（部署失败、上传失败、系统异常等）；

2. 日志查看：管理员可按日志类型、操作时间、操作用户、关键词等条件检索日志，查看日志详情（操作内容、操作时间、操作用户、IP地址、操作结果）；

3. 日志导出：支持将日志导出为Excel、CSV格式，便于后续分析和存档；

4. 日志清理：设置日志保留期限（如保留3个月、6个月），自动清理过期日志，也可手动清理指定时间段的日志。

## 3\.9 消息通知功能

1. 通知类型：
